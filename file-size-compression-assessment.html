Section A — Multiple Choice Questions (25 × 1)

1. What is the basic unit of data storage?

Answer: B — 8 bits.

Answer: C — KiB.
4. What is the formula for uncompressed bitmap image size (in bits)?



Answer: B — width × height × colour depth (bits).
5. What does 24-bit colour mean?



Answer: C — 8 bits per R,G,B channel.

Answer: B — ~605 MiB (textbook example for 60 min gives 605 MiB; for 60 s it's ~10.09 MiB; here answer B assumes long sample context). Note: students should show working.
7. Which compression is reversible so original can be reconstructed exactly?

Answer: B — 4 bits.

Answer: A — 19.

Answer: B — MSB 1 indicates negative in two's complement.

Section B — Short Answer Questions (10 × 2 = 20 marks)

1. Convert 123 to 8-bit binary. (2 marks)
Answer: 01111011
2. How many bytes are in 1 MiB? (2 marks)
Answer: 1,048,576 bytes
3. Calculate the size in MiB of an image 800 × 600 at 24 bpp. (show numeric answer) (2 marks)
Answer: 800×600×24 = 11,520,000 bits → bytes 1,440,000 → MiB ≈ 1.373291 (≈1.37)
4. What is the two's complement 8-bit representation of −112? (2 marks)
Answer: 10010000
5. If an audio file is sampled at 44,100 Hz, 16 bits, stereo, how many bits in one second? (2 marks)
Answer: 44,100 × 16 × 2 = 1,411,200 bits per second
6. Define ‘pixel’ in one sentence. (2 marks)
Answer: A pixel is the smallest addressable picture element in a digital image that stores colour/brightness information.
7. How many colours are possible with 8 bits per channel RGB? (2 marks)
Answer: 256^3 = 16,777,216 colours
8. Convert hexadecimal 3F to denary. (2 marks)
Answer: 63
9. Explain briefly why JPEG is called lossy. (2 marks)
Answer: JPEG discards information (especially colour detail) during compression, so the original cannot be reconstructed exactly.
10. A memory stick is 16 GiB. How many 5 MiB images can be stored? (2 marks)
Answer: 16 GiB = 16×1024 MiB = 16384 MiB; 16384 ÷ 5 = 3276 images (integer)

Section C — Scenario Questions (3 × 5 = 15 marks)

1. (5 marks) A game developer needs to store 150 background photos, each 4000×3000 at 24 bpp, and 20 voice clips each 2 minutes long at 44.1 kHz, 16-bit mono. She has a 256 GiB drive. Show by calculation whether the assets will fit. Show all workings and give a clear conclusion.
Model answer (summary):

One image: 4000×3000×24 = 288,000,000 bits = 36,000,000 bytes ≈ 34.33 MiB. 150 images ≈ 5,149.5 MiB ≈ 5.026 GiB. One voice clip: 44,100×16×1×120 = 84,672,000 bits = 10,584,000 bytes ≈ 10.09 MiB. 20 clips ≈ 201.8 MiB ≈ 0.197 GiB. Total ≈ 5.223 GiB << 256 GiB so YES they fit with large space left.

2. (5 marks) An audio producer must email a 600 MiB raw WAV file but the email limit is 25 MB. Suggest two precise steps (including a specific compression format and a change to audio parameters) to prepare the file, and estimate the new file size for each step. Explain trade-offs.
Model answer (summary):
  1. Encode to MP3 at 192 kbps: 600 MiB (~600×8=4800 Mbits) ÷ 192 kbps ≈ 25,000 seconds? Simpler: MP3 at 192 kbps ≈ 1.44 MB/min, so a 60-min file becomes ~86 MB — still too large. Use 128 kbps → ~57 MB. So to reach 25 MB need ~56 kbps, poor quality. Trade-off: lower quality.
  2. Change parameters: downsample to 22.05 kHz and 64 kbps MP3; estimated size ≈ (64 kbps × duration) → within 25 MB for typical lengths. Trade off: lower fidelity; acceptable for spoken word.
3. (5 marks) Describe a suitable compression strategy (lossy/lossless and why) for: (a) game sprite sheets with large flat colours; (b) background photographic textures in an online game; (c) final mastered movie soundtrack. Give one format for each and one sentence justification.
Model answer (summary):
  • (a) Sprites: Use lossless RLE or PNG (indexed) — exact colours needed, good compression on flat areas.
  • (b) Photographic textures: Use JPEG or WebP lossy with high quality — reduce file size and acceptable visual loss at distance.
  • (c) Mastered soundtrack: Use lossless (FLAC/WAV) for archive; use high-bitrate lossy (AAC/MP3 320 kbps) for distribution if needed.
